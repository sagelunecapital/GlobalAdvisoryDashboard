## Rules Applied

- **DEC-2026-04-18-01 (SPX Price = Daily High):** Column named `spx_daily_high` throughout. The `High` column from yfinance `^GSPC` is extracted (not `Close`). Tests assert `spx_price` and `spx_close` do not exist.
- **AC3 (Append-Only):** No `UPDATE` or `DELETE` SQL exists in any implementation file. Two tests (`test_no_update_or_delete_sql_in_historical_load`, `test_no_update_or_delete_sql_in_append`) grep source files at runtime to enforce this.
- **AC6 (Transactional Rollback):** `load_historical` uses explicit `conn.execute("BEGIN")` and `conn.execute("ROLLBACK")` — not Python's implicit transaction management — to guarantee clean state on failure.

## MMTH Data Source Decision

**Decision: EODData fallback is the active code path.**

- yfinance `^MMTH`: **UNAVAILABLE** as of 2026-04-22. Yahoo Finance returns HTTP 404 ("Quote not found for symbol: ^MMTH"). Ticker appears delisted or never supported.
- EODData API (`eodhistoricaldata.com`): Implemented as fallback in `src/fetch/mmth.py`. Requires `EODDATA_API_KEY` environment variable. If not set, raises `RuntimeError` with clear instructions to register and configure.
- Both `fetch_mmth_yfinance()` and `fetch_mmth_eoddata()` are implemented in `mmth.py`. `fetch_mmth()` (the public interface) tries yfinance first, then falls back to EODData.
- All tests use mocked MMTH data (`sample_mmth_series` fixture) — no live fetch required for CI.

**Action required before production use:** Set `EODDATA_API_KEY=<key>` or identify an alternative free MMTH source.

## Known Risks and Limitations

1. **MMTH source gap:** The EODData fallback requires a paid/free-tier API key. Until configured, `load_historical()` with live data will fail. Tests are fully isolated from this risk via fixtures.
2. **EMA warm-up:** The 2-year fetch for SPX is necessary to produce accurate EMAs. If `^GSPC` returns fewer than ~300 rows (e.g., on a new market), EMA values near the start will have reduced accuracy — standard EMA behavior.
3. **Inner-join date alignment:** MMTH and SPX may not share identical trading calendars on edge cases (holidays). The inner-join naturally handles this but could reduce row count slightly below 252 if alignment is poor. The `>= 252` guard will surface this as a `RuntimeError`.
4. **SQLite WAL and concurrent writes:** WAL mode allows concurrent readers but only one writer. This is sufficient for the dashboard use case (single-process refresh).
5. **`pytest-pylint` incompatibility:** On Python 3.13, `pytest-pylint` fails to import `PYLINTRC`. Suppressed via `pytest.ini` (`-p no:pylint`). Not a code issue.
