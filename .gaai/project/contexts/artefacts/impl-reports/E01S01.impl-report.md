## Known Risks / Limitations

- **`pd.isna()` on pandas Series scalar:** In Python 3.13 + pandas 2.x, `pd.isna(val)` on a scalar float returned by `.iloc[-1]` works correctly. If pandas ever returns a non-scalar (e.g., a Series due to duplicate index), the check would operate element-wise and the `if` condition would raise a ValueError. This is an unlikely edge case given the upstream `fetch_spx` guarantee of one row per date.
- **`math.isfinite` guard:** Handles `inf`/`-inf` values that pass the `pd.isna` check. Ensures no infinite floats reach the DB.
- **MMTH date alignment:** The last row of `mmth_series` may be from a different calendar date than the last row of `spx_df` if markets close on different days (e.g., bond holiday vs equity holiday). This is not addressed in AC scope; both sources use their own most-recent trading day. Alignment validation is deferred to a future story.
