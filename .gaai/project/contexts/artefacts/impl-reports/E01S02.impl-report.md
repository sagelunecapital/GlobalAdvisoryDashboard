## Rules Applied

- **DEC-2026-04-18-02:** Swing anchor is the pre-trough peak (for bearish) or the post-peak trough (for bullish), computed over the full 90-day window. Yesterday's high within the same run cannot become the anchor because the trough date is stable — tested by T9.
- **DEC-2026-04-22-01:** Only stdlib (`enum`, `sqlite3`) and `pandas` used; no new third-party dependencies.

## Algorithm Note

### Bearish anchor (prior swing HIGH)

1. Find the global minimum of `spx_daily_high` in the 90-day prior window — this is the correction trough that separated the prior bull run from the current run.
2. Take all rows strictly before the trough date ("pre-trough slice").
3. The prior swing HIGH is `idxmax()` of the pre-trough slice.
4. Edge case: if the trough is the earliest point in the window (no pre-trough rows), fall back to `idxmax()` of the full window.

Because the trough date is the structural minimum and is not displaced by new highs during the same run, the anchor is stable across all days in the same run — satisfying DEC-2026-04-18-02.

### Bullish anchor (prior swing LOW)

1. Find the global maximum of `spx_daily_high` in the 90-day prior window — this is the rally peak that preceded the current decline.
2. Take all rows on-or-after the peak date ("post-peak slice").
3. The prior swing LOW is `idxmin()` of the post-peak slice.
4. Edge case: if no post-peak rows exist, fall back to `idxmin()` of the full window.

### DATA_GAP guards (AC5)

Returned (not raised) in four cases, in priority order:
1. No row exists for `as_of_date` in the data.
2. Zero prior rows exist before `as_of_date`.
3. Earliest prior row is fewer than `SWING_LOOKBACK_DAYS` (90) calendar days before `as_of_date`.
4. MMTH is NaN at the identified swing anchor date.
