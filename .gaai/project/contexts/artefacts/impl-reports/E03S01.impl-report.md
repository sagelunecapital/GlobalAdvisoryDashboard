## AC Coverage Map

| AC | Implementation |
|---|---|
| AC1 — ≥52 weekly / ≥12 monthly observations | Enforced in `_series_to_records()` with `MIN_DAILY_ROWS=52` / `MIN_MONTHLY_ROWS=12` |
| AC2 — Monthly indicators store released value (no forecast) | `monthly_indicators.py` fetches FRED monthly levels; YoY/MoM computed from actuals |
| AC3 — GDPNow from Atlanta Fed, current quarter estimate | `gdpnow.py` hits Atlanta Fed Excel endpoint, returns most recent row |
| AC4 — Futures use continuous front-month contract prices | yfinance `GC=F`, `CL=F`, `HG=F`, `NG=F`, `ZC=F` are continuous front-month |
| AC5 — Subsequent refresh appends only; no overwrites/deletions | `INSERT OR IGNORE` on `(indicator_id, date)` PK; no UPDATE/DELETE anywhere |
| AC6 — Failed indicator identified; others continue | Per-indicator `try/except` in `_persist_series`; results dict shows `ok` or `error: <msg>` |
| AC7 — GDPNow failure isolated | GDPNow fetched in its own isolated `try/except` block; does not block other indicators |
| AC8 — Records include `indicator_id`, `date`, `value`, `fetch_timestamp` | Schema enforces all 4 columns as NOT NULL; `fetch_timestamp` = UTC ISO-8601 |
| AC9 — Consistent named indicator IDs for E03S02 | All 32 IDs in `ALL_INDICATOR_IDS`; stored with `indicator_id`, `date`, `value` fields |
