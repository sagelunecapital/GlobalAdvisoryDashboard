# Memory Delta — E01S06: Initialize and Maintain Historical Indicator Database

## New Patterns Established

### SQLite with WAL Mode
- All database connections use `PRAGMA journal_mode=WAL` for ACID-safe concurrent reads.
- Connection factory (`get_connection`) is the single entry point; callers never open `sqlite3.connect()` directly in production code.
- `isolation_level` is left at default (not `None`); callers use explicit `BEGIN`/`COMMIT`/`ROLLBACK` for transactional bulk loads and `conn.commit()` for single-row appends.

### Append-Only Pattern
- All writes use `INSERT OR IGNORE` — duplicate dates are silently skipped, never overwritten.
- No `UPDATE` or `DELETE` SQL is present anywhere in the production codebase.
- Schema enforces `date TEXT PRIMARY KEY` to make the append-only constraint enforceable at the database level.

### Transactional Rollback for Bulk Loads
- `load_historical()` issues an explicit `conn.execute("BEGIN")` before any inserts.
- Any exception triggers `conn.execute("ROLLBACK")` in the except block, ensuring zero partial data is persisted.
- The raised `RuntimeError` message explicitly confirms the DB was rolled back and the operation is retryable.
- This pattern (explicit transaction + rollback on failure + retryable error message) is the standard for all future bulk data loads.

### Test Injection Pattern for Live Data Dependencies
- Functions that call live APIs accept optional `_param` overrides (e.g., `_spx_df`, `_mmth_series`) for test injection.
- This avoids mocking at the module level and keeps tests deterministic without network calls.

## New Conventions

### Column Naming (DEC-2026-04-18-01)
- SPX price column is `spx_daily_high` — always the daily HIGH price, never `spx_price`, `spx_close`, or `spx_open`.
- This decision is enforced in schema DDL, all INSERT SQL strings, fetch module output, and test assertions.

### Date Format
- All date values stored as ISO-8601 strings (`YYYY-MM-DD`) in the `date TEXT PRIMARY KEY` column.
- Pandas date columns are formatted via `.dt.strftime("%Y-%m-%d")` before database insertion.

## MMTH Data Source Finding

- **yfinance `^MMTH`: UNAVAILABLE as of 2026-04-22.** Yahoo Finance returns HTTP 404 for this ticker. The `fetch_mmth_yfinance()` function documents this and raises immediately on empty response.
- **EODData API is the active MMTH data source.** Endpoint: `https://eodhistoricaldata.com/api/eod/MMTH.INDX`. Requires `EODDATA_API_KEY` environment variable.
- If `EODDATA_API_KEY` is not set, `fetch_mmth_eoddata()` raises a `RuntimeError` with registration URL and retry instructions — no silent failure.
- **Production prerequisite:** Confirm EODData free tier provides 252+ trading days of MMTH.INDX history before go-live. This was not verified with a live API call during E01S06 delivery (tests use injected mock data).
- **Future watch:** Monitor Yahoo Finance for `^MMTH` re-listing; the yfinance path in `fetch_mmth()` will automatically take over if the ticker becomes available again (it checks `len(series) >= 252`).
