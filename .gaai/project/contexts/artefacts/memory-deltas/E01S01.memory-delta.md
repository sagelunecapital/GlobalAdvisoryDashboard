- candidate_id: CANDIDATE-001
  category: patterns
  description: >
    Dependency injection via private underscore parameters (_spx_df, _mmth_series) is the established pattern for making
    external-data-fetching functions testable without mocking at the module level. fetch_live() accepts optional override
    parameters that bypass live network calls in tests. This keeps test fixtures in conftest.py and test logic in test
    files without needing monkeypatch for the happy-path tests.
  suggested_tags: [testing, dependency-injection, fetch-layer, patterns]
  ingestion_priority: medium

- candidate_id: CANDIDATE-002
  category: patterns
  description: >
    The fetch orchestrator pattern: each external source is wrapped in an independent try/except, errors are accumulated
    in a list, and a single FetchError is raised after all sources have been attempted. This ensures partial-failure
    reporting (all failing sources named in one exception) without aborting on the first failure. Established in
    src/fetch/live.py and should be the template for future fetch orchestrators (e.g., Liquidity module).
  suggested_tags: [error-handling, fetch-layer, patterns, resilience]
  ingestion_priority: high

- candidate_id: CANDIDATE-003
  category: decisions
  description: >
    MMTH date alignment is NOT validated against SPX date in fetch_live(). The implementation takes iloc[-1] from each
    source independently — if they correspond to different trading dates (e.g., bond market holiday vs equity holiday),
    the mismatch is silently accepted. This is an acknowledged limitation noted in the impl-report; alignment validation
    is deferred to a future story. Discovery should decide whether to add an AC to a future story or accept the risk.
  suggested_tags: [mmth, spx, data-alignment, deferred, risk]
  ingestion_priority: medium
