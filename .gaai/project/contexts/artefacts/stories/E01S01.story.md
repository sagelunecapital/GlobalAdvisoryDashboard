# Story E01S01: Fetch Live Indicator Data

## User Story

**As a** fund manager,
**I want** the dashboard to automatically retrieve current values for SPX price, SPX 12-day EMA, SPX 25-day EMA, and MMTH,
**so that** I have current indicator data each time I access the dashboard without manual lookup.

---

## Context

The market regime classification depends on four indicator values. These must be fetched from reliable data sources automatically. MMTH (% of all US stocks above their 200-day MA) is confirmed available on barcharts.com. SPX price and EMA values require a suitable data source to be selected during Delivery — Delivery may fetch pre-calculated EMA values or calculate them from raw OHLCV data; either approach is acceptable.

---

## Acceptance Criteria

- [ ] AC1: Given the dashboard loads, the system fetches the current SPX **daily high** price
- [ ] AC2: Given the dashboard loads, the system fetches or computes the current SPX 12-day EMA value
- [ ] AC3: Given the dashboard loads, the system fetches or computes the current SPX 25-day EMA value
- [ ] AC4: Given the dashboard loads, the system fetches the current MMTH value (% US stocks above 200-day MA) — barcharts.com is a confirmed source
- [ ] AC5 (error-handling): Given one or more indicator fetches fail, the system surfaces a visible error message identifying which indicator(s) failed to load — it does not display partial data as if the full set were available, and it does not crash
- [ ] AC6 (error-handling): Given all four indicator fetches succeed but one value is missing or null, the system treats the full fetch as failed and surfaces the same visible error — no regime classification proceeds from an incomplete indicator set
- [ ] AC7: Each successful daily fetch is persisted to the historical indicator database — data is appended, never overwritten or deleted

---

## Out of Scope

- User-configurable data sources
- Database schema design or table creation (handled in E01S06)

---

## Dependencies

- E01S06 — historical database must exist before fetched data can be persisted (AC7)

---

## Notes

Delivery has full freedom to choose data sources for SPX, EMA, and MMTH values. EMA may be fetched as a pre-calculated value or computed client-side from OHLCV data — Delivery decides. MMTH is available on barcharts.com; Delivery should confirm whether a programmatic API or scraping approach is viable, or flag if a different source is needed.
