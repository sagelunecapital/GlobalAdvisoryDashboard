# Story E01S02: Detect SPX / MMTH Divergence

## User Story

**As a** fund manager,
**I want** the system to automatically detect whether SPX and MMTH are in bearish or bullish divergence,
**so that** divergence conditions are accurately factored into the regime classification without manual assessment.

---

## Context

Divergence is defined as: SPX makes a new high (or new low) but MMTH does not confirm with a corresponding new high (or new low).

- **Bearish divergence:** SPX makes a new high, but MMTH at that point is lower than at the prior SPX swing high.
- **Bullish divergence:** SPX makes a new low, but MMTH at that point is higher than at the prior SPX swing low (MMTH did not follow price down).

A "prior swing high" is the most recent identifiable SPX **daily high** peak before the current run; a "prior swing low" is the most recent identifiable SPX trough before the current run. Delivery defines and documents the swing detection algorithm (lookback window, peak/trough identification method). QA verifies divergence detection correctness against Delivery's documented algorithm — not against an undeclared expected output.

**Divergence period anchor rule:** All SPX daily highs within the same run (consecutive days that all exceed the same prior swing high) are compared against that same prior swing high — they are NOT compared against each other. Yesterday's high within the same run is never the anchor; only the prior period's swing high is the anchor.

**Example (bearish):** SPX high Apr 15, 2026 = 7,026 vs prior swing high Jan 28, 2026 = 7,002 (SPX made new high). MMTH Apr 15 = 54.99 vs MMTH Jan 28 = 61.50 (MMTH lower). SPX new high unconfirmed by MMTH → bearish divergence.

---

## Acceptance Criteria

- [ ] AC1: Given SPX **daily high** has exceeded its most recent prior swing high AND MMTH at the current date is lower than MMTH at the prior swing high date → the system flags a bearish divergence; the prior swing high remains the anchor for all subsequent days in the same run until a new prior swing high is established
- [ ] AC2: Given SPX **daily high** has fallen below its most recent prior swing low AND MMTH at the current date is higher than MMTH at the prior swing low date → the system flags a bullish divergence; the prior swing low remains the anchor for all subsequent days in the same run
- [ ] AC3: Given neither bearish nor bullish condition is met → the system returns no divergence
- [ ] AC4: The system reads from the historical indicator database (populated by E01S06 with at least 1 year of data) to identify prior swing highs and lows — the database is the authoritative source for all historical comparison
- [ ] AC5 (error-handling): Given insufficient historical data to identify a prior swing high or low, the system returns a DATA_GAP signal (distinct from the NO_DIVERGENCE signal returned by AC3) and surfaces a user-visible data-gap indicator — downstream consumers (e.g., E01S03) must treat DATA_GAP as an unavailable input, not as a clean no-divergence state — it does not produce a false divergence signal

---

## Out of Scope

- Multi-swing or extended divergence patterns (only the most recent prior swing is compared)
- Divergence charting or visual annotation (display handled in E01S04)
- User-adjustable lookback windows

---

## Dependencies

- E01S01 — indicator data must be available before divergence can be evaluated
- E01S06 — historical database must be populated before prior swing highs/lows can be identified (AC4)

---

## Notes

Delivery defines the swing detection algorithm (lookback window, identification method) and documents it. The ACs define behavioral outcomes; the algorithm is Delivery's decision. The "one step max" regime-shift constraint from the "price is king" principle is enforced by the regime classification layer (E01S03), not by this story.

The data lookback required by AC4 (operational window for swing detection) is distinct from the S-2 constraint ("no historical regime tracking"). AC4 retains price/MMTH data for detection computation — it does not log or track historical regime states.
