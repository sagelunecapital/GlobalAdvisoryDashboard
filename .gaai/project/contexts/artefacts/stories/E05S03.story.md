# Story E05S03: Display Regime Section on Macro Tab

## User Story

**As a** fund manager,
**I want** a "Regime" section at the top of the Macro tab showing all four criteria, Market Narrative, and Long Bias,
**so that** I have a single synthesised positioning read before I scroll into the detailed macro indicator tables.

---

## Context

The Macro tab (established by E03S04) shows a categorised table of 32 macro indicators. The Regime section is a new block inserted above all E03S04 indicator sections — at the top of the Macro tab, before any categorized indicator rows. It reads the pre-computed regime state persisted by E05S02 and renders it without re-computation.

---

## Acceptance Criteria

- [ ] AC1: A section labelled "Regime" renders at the top of the Macro tab, above all E03S04 indicator sections (before any categorized indicator rows — Inflation, Yields, Currencies, Futures, Growth).
- [ ] AC2: The section displays all four criteria (Growth, Inflation, Monetary Policy Stance, Global Credit Creation) with their computed labels exactly as stored by E05S02.
- [ ] AC3: The GCC sub-section shows all six individual checklist item labels (Positive Impulse, Negative Impulse, or Unavailable) followed by the aggregate GCC verdict.
- [ ] AC4: Market Narrative and Long Bias are displayed within the Regime section.
- [ ] AC5 (data-freshness): The `as_of_date` read from the persisted regime state (computed by E05S02 per DEC-2026-04-18-03) is displayed alongside the Regime section — not re-derived from the current calendar date.
- [ ] AC6 (error-handling): If any individual criterion is "Unavailable" in the persisted state, that criterion renders "Unavailable" — the rest of the section continues to display normally.
- [ ] AC7 (error-handling): If the persisted regime state file/table cannot be read (missing, corrupt, or stale), the Regime section renders an explicit error state ("Regime data unavailable — run refresh") rather than a blank or partial section.
- [ ] AC8 (signal-detection): All stored criterion labels — including "Unavailable", "Unclear", and any of the named regime labels — are rendered verbatim from the persisted state. The display layer does not substitute, remap, or normalize any label.

---

## Out of Scope

- Re-computing regime signals at display time — reads pre-computed state from E05S02 only
- Charting historical regime states
- Alerts or notifications when regime changes
- Updating the dashboard prototype — handled in E05S04

---

## Dependencies

- E05S02 — regime state must be computed and persisted before the display layer can read it
- E03S04 — Macro tab display must exist as the container into which the Regime section is inserted
