# Story E01S03: Classify Market Regime

## User Story

**As a** fund manager,
**I want** the system to classify the current market regime as Red, Yellow, or Green based on all validated conditions,
**so that** I have a consistent, rule-based regime read every day.

---

## Context

Regime classification uses SPX price position relative to its 12-day and 25-day EMAs, combined with the divergence signal from E01S02 (bearish / bullish / none / DATA_GAP). Nine input combinations cover all valid states.

**EMA zone boundaries:** SPX ≥ 12d EMA = "above" zone; SPX < 12d EMA and SPX ≥ 25d EMA = "between" zone; SPX < 25d EMA = "below" zone. When SPX equals an EMA exactly, the ≥ side takes precedence (above beats between; between beats below).

The "price is king" principle: EMA position is the baseline regime anchor. A divergence signal shifts the regime by one step at most — never two. This means:
- Bearish divergence downgrades Green → Yellow; it cannot downgrade Yellow → Red when price is in the above zone.
- Bullish divergence upgrades Red → Yellow; it cannot upgrade Yellow → Green.
- When SPX is between the EMAs (baseline = Yellow), bullish divergence has no higher step — the signal is absorbed and regime stays Yellow.
- When SPX is above 12d EMA (baseline = Green), bullish divergence has no higher step — the signal is absorbed and regime stays Green.
- Bearish divergence from between-EMA state overrides to Red — directionally consistent with downtrend risk.

The classification output includes both the regime label (Green / Yellow / Red) and the active condition identifier (which of the 9 conditions produced the result), so the display layer (E01S04) can show what's driving the regime.

---

## Acceptance Criteria

- [ ] AC1: Given SPX ≥ 12d EMA and divergence signal = none → regime = Green, condition = 1
- [ ] AC2: Given SPX ≥ 12d EMA and divergence signal = bearish → regime = Yellow, condition = 2
- [ ] AC3: Given SPX ≥ 12d EMA and divergence signal = bullish → regime = Green, condition = 3 (bullish divergence absorbed — no step above Green)
- [ ] AC4: Given SPX is between EMAs and divergence signal = none → regime = Yellow, condition = 4
- [ ] AC5: Given SPX is between EMAs and divergence signal = bullish → regime = Yellow, condition = 5 (bullish divergence absorbed — no step above Yellow from between-EMA)
- [ ] AC6: Given SPX is between EMAs and divergence signal = bearish → regime = Red, condition = 6
- [ ] AC7: Given SPX < 25d EMA and divergence signal = bullish → regime = Yellow, condition = 7
- [ ] AC8: Given SPX < 25d EMA and divergence signal = none → regime = Red, condition = 8
- [ ] AC9: Given SPX < 25d EMA and divergence signal = bearish → regime = Red, condition = 9
- [ ] AC10 (error-handling): Given any required indicator value (SPX price, 12d EMA, 25d EMA) is missing, or the divergence signal from E01S02 is unavailable or DATA_GAP → regime classification is not computed; the system returns an explicit unclassified state — no regime label or condition identifier is produced from incomplete inputs

## Notes

AC6 (between EMAs + bearish) and AC5 (between EMAs + bullish) are theoretically valid but practically rare (O-1). The classification logic must handle all 9 combinations correctly regardless. DATA_GAP from E01S02 is treated identically to an unavailable signal — AC10 applies.
