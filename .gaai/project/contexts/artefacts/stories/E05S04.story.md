# Story E05S04: Update Dashboard Prototype to Include Regime Section

## User Story

**As a** fund manager,
**I want** the static HTML dashboard prototype to include a representative Regime section on the Macro tab,
**so that** I can review the full information hierarchy — including the cross-signal section and its edge-case states — before the live pipeline is wired up.

---

## Context

The dashboard prototype (`prototypes/index.html`) is a single-file, self-contained HTML file with hardcoded sample data (established by E04S01). This story adds a hardcoded Regime section to the Macro tab in that prototype, positioned above all E03S04 indicator sections, matching the live display spec delivered by E05S03. Delivery must inspect the completed E05S03 implementation before building the prototype — "matching the live display spec" means mirroring E05S03's delivered code, not an abstract spec document. No live data or backend connection is introduced — all values are hardcoded sample data.

---

## Acceptance Criteria

- [ ] AC1: `prototypes/index.html` is updated to include a "Regime" section on the Macro tab, positioned above all existing indicator sections (above the first categorized indicator row established by E03S04).
- [ ] AC2: A primary hardcoded regime state displays all four criteria with representative sample labels (e.g., "Growth: Accelerating", "Inflation: Accelerating", "Monetary Policy Stance: Bear Steepener", "Global Credit Creation: Negative Impulse").
- [ ] AC3: The GCC sub-section shows all six hardcoded checklist items (SOFR > IORB, Net Liquidity, USDJPY Falling, USDCNH Rising, EURCHF Falling, DXY Rising) with their Positive/Negative Impulse labels.
- [ ] AC4: Market Narrative and Long Bias are shown with hardcoded sample values consistent with the primary state (e.g., "Market Narrative: Reflation", "Long Bias: Defensives").
- [ ] AC5: An `as_of_date` is displayed with the Regime section using a hardcoded sample trading session date (a Friday) consistent with the rest of the prototype's sample dates, per DEC-2026-04-18-03.
- [ ] AC6: Visual design of the Regime section is consistent with the existing prototype theme — background `#101419`, cards `#1c2026`, accent `#0070cc` (PlayStation Blue), Inter weight 300 for display headlines, card border-radius 12–24px.
- [ ] AC7: The prototype file continues to open correctly from `file://` protocol with no server required — no new external dependencies introduced.
- [ ] AC8 (signal-detection): The prototype includes a second hardcoded example within the Regime section (e.g., as a commented-out block or a visible secondary state) demonstrating "Long Bias: Unclear" (e.g., MPS: Neutral or MPS: Steepener Twist) so stakeholders can review the full label vocabulary.
- [ ] AC9 (error-handling): The prototype includes a hardcoded example (e.g., as a commented-out block or a visible secondary state) demonstrating at least one criterion rendered as "Unavailable", so stakeholders can review the error-display path.

---

## Out of Scope

- Live data integration — hardcoded sample data only
- Modifying the Breadth or Execution tabs
- Changing any existing Macro tab section beyond inserting the Regime block

---

## Dependencies

- E05S03 — must be fully delivered before E05S04 begins; Delivery mirrors E05S03's implemented component, not a paper spec

---

## Notes

For AC2/AC4 primary sample: "Growth: Accelerating, Inflation: Accelerating, MPS: Bear Steepener, GCC: Negative Impulse → Market Narrative: Reflation, Long Bias: Defensives" is consistent with E05S02's rules and exercises a complete display path. For AC8 Unclear example: "MPS: Neutral → Long Bias: Unclear" is the simplest case to hardcode. For AC9 Unavailable example: showing Growth: Unavailable (GDPNow missing) demonstrates the error path. Delivery may implement AC8 and AC9 as collapsed comment blocks or a small "display states" section styled as a reference panel — the goal is stakeholder visibility, not production fidelity.
