# Story E05S02: Compute Regime Signals

## User Story

**As a** fund manager,
**I want** the dashboard to automatically compute the four Regime criteria and two derived outputs from the available DB data,
**so that** I receive a single, deterministic positioning read without manually interpreting multiple indicators.

---

## Context

All four Regime criteria draw from existing DB tables (E02S02: Net Liquidity; E03S01: GDPNow raw level; E03S02: WoW% changes for T5YIE, US02Y, US10Y, DXY, USDJPY, USDCNH) plus the new series stored by E05S01 (SOFR_WEEKLY, IORB_WEEKLY, EURCHF_5D_PERF, GDP_YOY_PCT). This story reads those tables, applies the classification rules, and persists the computed regime state for consumption by the display layer (E05S03).

**Classification rules (canonical — must be implemented exactly as specified):**

### Growth
- Latest `GDPNow` value (raw level from E03S01 DB) > latest `GDP_YOY_PCT` value (from E05S01 DB) → "Growth: Accelerating"
- Otherwise → "Growth: Decelerating"

### Inflation
- Latest `T5YIE` WoW% change (from E03S02 DB) > 0 → "Inflation: Accelerating"
- Otherwise → "Inflation: Decelerating"

### Monetary Policy Stance (MPS) — uses WoW% change of US02Y and US10Y from E03S02
| Condition | Label |
|---|---|
| Chg_US02Y < Chg_US10Y < 0 | Monetary Policy Stance: Bull Steepener |
| 0 < Chg_US02Y < Chg_US10Y | Monetary Policy Stance: Bear Steepener |
| Chg_US10Y < Chg_US02Y < 0 | Monetary Policy Stance: Bull Flattener |
| 0 < Chg_US10Y < Chg_US02Y | Monetary Policy Stance: Bear Flattener |
| Chg_US02Y < 0 < Chg_US10Y | Monetary Policy Stance: Steepener Twist |
| Chg_US10Y < 0 < Chg_US02Y | Monetary Policy Stance: Flattener Twist |
| None of the above (both zero, equal non-zero, or any combination not matching the 6 cases above) | Monetary Policy Stance: Neutral |

### Global Credit Creation (GCC) — 6-item checklist
Each item produces either "Negative Impulse" (condition TRUE) or "Positive Impulse" (condition FALSE). Strict inequality operators govern FX directional items — a flat WoW% change of exactly 0 does NOT satisfy a negative condition and defaults to Positive Impulse.

**Item 1 (level comparison — not WoW%):** `SOFR_WEEKLY` > `IORB_WEEKLY` (latest raw values) → "SOFR > IORB: Negative Impulse"
**Item 2:** Net Liquidity WoW% < 0 → "Net Liquidity: Negative Impulse"
**Item 3:** `USDJPY` WoW% change (from E03S02) < 0 → "USDJPY Falling: Negative Impulse"
**Item 4:** `USDCNH` WoW% change (from E03S02) > 0 → "USDCNH Rising: Negative Impulse"
**Item 5:** `EURCHF_5D_PERF` value (from E05S01) < 0 → "EURCHF Falling: Negative Impulse"
**Item 6:** `DXY` WoW% change (from E03S02) > 0 → "DXY Rising: Negative Impulse"

**Aggregate verdict (available items only — exclude items marked "Unavailable" from the count):**
- Negative count > Positive count → "Global Credit Creation: Negative Impulse"
- Negative count < Positive count → "Global Credit Creation: Positive Impulse"
- Tie (equal neg and pos among available items): recount using items 1, 2, 6 that are available only
  - If items 1/2/6 are available and give a clear majority → use that
  - If items 1/2/6 also tie or ≤1 of them is available → "Global Credit Creation: Unavailable"

### Market Narrative
| Growth | Inflation | Label |
|---|---|---|
| Accelerating | Accelerating | Market Narrative: Reflation |
| Accelerating | Decelerating | Market Narrative: Goldilocks |
| Decelerating | Accelerating | Market Narrative: Stagflation |
| Decelerating | Decelerating | Market Narrative: Deflation |

### Long Bias
The lookup uses the criterion label content (not the stored prefix "Monetary Policy Stance:"):

| MPS label content | GCC | Long Bias label |
|---|---|---|
| Bull Flattener or Bull Steepener | Positive Impulse | Long Bias: High Beta |
| Bull Flattener or Bull Steepener | Negative Impulse | Long Bias: Secular |
| Bear Flattener or Bear Steepener | Positive Impulse | Long Bias: Real Economy (Cyclicals) |
| Bear Flattener or Bear Steepener | Negative Impulse | Long Bias: Defensives |
| Steepener Twist, Flattener Twist, or Neutral | any | Long Bias: Unclear |

## Acceptance Criteria

- [ ] AC1 (signal-detection): Growth signal computed from latest raw `GDPNow` level (E03S01 DB) vs latest `GDP_YOY_PCT` (E05S01 DB); result is "Growth: Accelerating" or "Growth: Decelerating".
- [ ] AC2 (signal-detection): Inflation signal computed from latest `T5YIE` WoW% change (E03S02 DB); result is "Inflation: Accelerating" or "Inflation: Decelerating".
- [ ] AC3 (signal-detection): MPS computed from latest WoW% changes of `US02Y` and `US10Y` (E03S02 DB); result is one of the 7 labels in the classification table (including Neutral for edge cases).
- [ ] AC4 (signal-detection): GCC computed from all 6 checklist items per the rules in Context. Item 1 uses a raw level comparison (`SOFR_WEEKLY` > `IORB_WEEKLY`) — not WoW%. Items 3–6 use WoW% or performance values per item definitions. Each item's individual label ("Negative Impulse", "Positive Impulse", or "Unavailable") is persisted alongside the aggregate verdict.
- [ ] AC5 (signal-detection): Net Liquidity WoW% derived from the two most recent observations in the E02 Net Liquidity DB table: `(latest − prior) / abs(prior) × 100`. If fewer than two observations exist, GCC item 2 is marked "Unavailable" and excluded from the aggregate count. In a tie-break scenario, if item 2 is Unavailable, the tie-break uses items 1 and 6 only; if those also tie 1-1, GCC is "Unavailable".
- [ ] AC6 (signal-detection): Market Narrative derived from Growth × Inflation matrix; result is one of the four labels (Reflation, Goldilocks, Stagflation, Deflation).
- [ ] AC7 (signal-detection): Long Bias derived from MPS × GCC matrix using the label content (strip the "Monetary Policy Stance:" prefix before lookup); result is one of the five labels including "Unclear" for Twist and Neutral MPS.
- [ ] AC8 (data-freshness): Computed regime state includes an `as_of_date` field. When all weekly input series have the same latest Friday date, that date is used. When sources differ, `as_of_date` = the most recent Friday for which all weekly series (E03S02, E02S02, E05S01 SOFR/IORB/EURCHF) have observations — not the current calendar date, per DEC-2026-04-18-03.
- [ ] AC9 (error-handling): If any individual item's input value is NULL or missing, that GCC item is "Unavailable" and excluded from the aggregate count. If any other individual input (T5YIE WoW%, US02Y WoW%, US10Y WoW%, GDPNow, GDP_YOY_PCT) is NULL, the dependent criterion is set to "Unavailable" — it does not block other criteria from computing.
- [ ] AC10 (error-handling): If MPS is "Unavailable" (missing US02Y or US10Y data, not the same as Neutral), Long Bias is also set to "Unavailable" (not "Unclear"). "Unavailable" (missing data) and "Unclear" (MPS is Twist or Neutral but data was present) are distinct states.
- [ ] AC11: Computed regime state is persisted (DB table or JSON file at a stable path) with fields for each criterion label, each GCC item label, Market Narrative, Long Bias, and `as_of_date`. A re-run overwrites the current state (no historical regime accumulation). Fields sufficient for E05S03 to read without re-computation.

- Historical regime state tracking (only current state is computed and persisted)

## Notes

USDJPY, USDCNH, and DXY directional signals use the WoW% change columns computed by E03S02 — do not re-fetch from barchart. EURCHF_5D_PERF is the 5-day performance value stored by E05S01 (consumed directly as-is). Net Liquidity WoW% is derived by this story from the E02 Net Liquidity table; E02S03 computes YoY% and 3MA but not WoW%. The Long Bias lookup strips the "Monetary Policy Stance:" prefix from the stored MPS label before comparing against the matrix key column — Delivery must implement accordingly to avoid string-match failures.
