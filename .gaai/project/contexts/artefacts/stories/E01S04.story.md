# Story E01S04: Display Regime and Indicators

## User Story

**As a** fund manager,
**I want** to see the current market regime displayed as the primary element of the module alongside the supporting indicator values, the active condition driving it, and divergence context,
**so that** I can read the regime and understand what's behind it — not just the label.

---

## Context

Displaying only the regime label (Red/Yellow/Green) is insufficient — fund managers need to see what's driving the classification (Q-2). Supporting values, the active condition, and divergence context must be co-located with the regime label. The display must support cohesive decision-making (Q-1), which requires making the bounded nature of divergence visible — a fund manager must know whether a divergence has already shifted the regime or is merely absorbed.

---

## Acceptance Criteria

- [ ] AC1: The current regime is displayed as the primary visual element of the regime module, color-coded using the mandated regime colors: Green for Green, Yellow for Yellow, Red for Red
- [ ] AC2: The four indicator values are displayed: SPX daily high price, SPX 12d EMA, SPX 25d EMA, MMTH value
- [ ] AC3: The active classification condition is displayed as a human-readable description (e.g., "SPX above 12d EMA — no divergence"; not an opaque code or number)
- [ ] AC4: The divergence state is always explicitly labeled — if a divergence is active, its type (bearish or bullish) is shown with the regime effect (e.g., "bearish divergence — regime downgraded from Green to Yellow"); if no divergence is active, this is explicitly shown (e.g., "no divergence") rather than left blank
- [ ] AC5: The **trading session date** (the date of the indicator data, not the calendar date the fetch was triggered) and the timestamp of the most recent fetch are both displayed — e.g., if data is fetched on a Saturday for the previous Friday's session, the session date shown is Friday, not Saturday
- [ ] AC6 (error-handling): If the regime is in an unclassified state due to missing indicator data, an explicit error state is displayed — no regime color label is shown, and the specific missing indicator (e.g., "SPX data unavailable") is named
- [ ] AC7 (error-handling): If the regime is unclassified because the data source failed entirely, the error message is distinct from AC6 (e.g., "Data source unreachable" vs "SPX data unavailable") — these two error states use different labels so the fund manager can distinguish a source failure from a missing-value failure

---

## Out of Scope

- Historical regime chart or timeline
- Animated transitions or real-time streaming updates (handled in E01S05)
- Mobile layout optimization

---

## Dependencies

- E01S03 — regime classification result, active condition identifier, and divergence signal required before display

---

## Notes

The regime color names (Green / Yellow / Red) are required information requirements. The specific shade, saturation, and visual treatment within each color is Delivery's decision. Layout, typography, and component design are Delivery's decision. The ACs define what information must be present and human-readable, not how it must be styled.

AC5 timestamp is derived from Q-1 and Q-2 — a fund manager cannot assess data reliability without knowing when it was last fetched.
