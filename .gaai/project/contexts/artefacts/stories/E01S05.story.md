# Story E01S05: Refresh Indicator Data

## User Story

**As a** fund manager,
**I want** the dashboard to automatically refresh indicator data once each morning and allow me to manually trigger a refresh at any time,
**so that** I have a current regime read at the start of each trading day and can update it on demand when needed.

---

## Context

Fund managers review market regime at the start of the trading day. An automatic 8:00 AM Manila time (UTC+8) refresh ensures data is current when they open the dashboard. A manual refresh control provides on-demand updates without waiting for the scheduled trigger.

---

## Acceptance Criteria

- [ ] AC1: After a manual refresh completes successfully, the displayed regime, indicator values, and timestamp all reflect the newly fetched data
- [ ] AC2: The dashboard auto-refreshes once per day at 8:00 AM Philippine Standard Time (UTC+8)
- [ ] AC3: The timestamp of the last successful refresh is always visible and updates after each successful refresh
- [ ] AC4 (error-handling): If a refresh attempt fails, the previously fetched data remains displayed with a visible staleness warning — the dashboard does not go blank or show an empty state
- [ ] AC5 (error-handling): After 3 or more consecutive failed refresh attempts, a persistent error indicator replaces the staleness warning — it remains displayed until the next successful refresh

---

## Out of Scope

- Push-based real-time streaming
- User-configurable refresh intervals at runtime
- Selective refresh of individual indicators

---

## Dependencies

- E01S01 — refresh triggers a new data fetch; the fetch layer must exist
- E01S02 — divergence detection re-runs after each successful refresh
- E01S03 — regime reclassification re-runs after each successful refresh
- E01S04 — display must exist before refresh behavior is meaningful

---

## Notes

Delivery determines the auto-refresh mechanism (scheduled job, cron, background worker, etc.). The 8:00 AM UTC+8 trigger is a business requirement. New indicator values fetched on each refresh are appended to the historical database per E01S06 — no data is discarded.
