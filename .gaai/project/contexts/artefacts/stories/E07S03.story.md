# Story: Build gaai doctor Pre-Flight Command

## User Story

**As a** developer about to run delivery on Windows,
**I want** `gaai_deliver.py --doctor` to check my environment and auto-fix known issues before launch,
**so that** I discover environment problems before they cause a delivery failure mid-session.

---

## Context

Multiple delivery sessions have been lost to environment issues discovered mid-flight: missing `skipDangerousModePermissionPrompt` causes interactive prompts that hang forever in headless mode; stale PID files falsely indicate a running daemon; missing lock directories crash startup. The `--doctor` command codifies the known failure modes into a pre-flight checklist, auto-fixes what it can, and exits 1 with a clear report if a blocking issue cannot be auto-fixed.

`health-check.sh` (the existing bash health check) is itself a bash script that fails on Windows — `--doctor` is its Windows-compatible replacement for environment-level checks.

---

## Acceptance Criteria

- [ ] AC1: `gaai_deliver.py --doctor` checks whether `skipDangerousModePermissionPrompt: true` is set in the file at `CLAUDE_SETTINGS_PATH` (a module-level constant in `gaai_deliver.py` set to `Path.home() / '.claude' / 'settings.json'`). If absent: adds the key and reports "Fixed: added skipDangerousModePermissionPrompt". If present: reports "OK: skipDangerousModePermissionPrompt". If the file exists but cannot be parsed as JSON (malformed or non-JSON content), reports "ERROR: could not parse {CLAUDE_SETTINGS_PATH}: {reason}" and does not attempt to add the key — a malformed settings file must be corrected manually. If the file does not exist, creates it as `{"skipDangerousModePermissionPrompt": true}` and reports "Fixed: created settings.json and added skipDangerousModePermissionPrompt".
- [ ] AC2: `gaai_deliver.py --doctor` checks whether `.delivery-locks/` directory exists. Creates it if absent using `os.makedirs()` and reports "Fixed: created .delivery-locks/". If `os.makedirs()` raises an exception (e.g., permission denied), reports "ERROR: could not create .delivery-locks/: {reason}" — this triggers AC3's skip condition ("ERROR: .delivery-locks/ missing — cannot check PID files"). If the directory is already present: reports "OK: .delivery-locks/ exists". This check runs before AC3 — the directory must exist before the stale PID scan.
- [ ] AC3: `gaai_deliver.py --doctor` checks whether `.delivery-locks/` contains stale PID files. PID file format: files with a `.pid` extension; the filename stem is the story ID (e.g., `E07S03.pid`); the file content is the process PID as a decimal integer string — only the first line of the file is read (via `content = file.readline().strip()`; trailing whitespace and newlines are stripped). Only `.pid`-extension files are scanned — other files in the directory are ignored. A file is stale if: (a) `int(content)` raises `ValueError` (content cannot be parsed as a decimal integer after stripping), OR (b) `psutil.pid_exists(int(content))` returns `False`. Stale files are removed. Reports "Fixed: removed N stale PID file(s)". If no `.pid` files are stale: reports "OK: no stale PID files". If a `.pid` file content cannot be parsed as an integer, the file is treated as stale and removed (not an ERROR). If `os.remove()` fails for any stale file (e.g., permission denied), the failure is reported as "WARN: could not remove {filename}: {reason}" and the scan continues to remaining files — the final fixed count reflects only files successfully removed. Runs after AC2 — if AC2's directory creation failed (and the directory still does not exist), AC3 reports "ERROR: .delivery-locks/ missing — cannot check PID files" and skips the scan.
- [ ] AC4: `gaai_deliver.py --doctor` checks for a Python executable in PATH using `shutil.which('python')` first, then `shutil.which('python3')` if the first returns `None`. (`shutil.which` must be used — not a subprocess invocation — to avoid the Windows Microsoft Store redirect, where `python` in a subprocess may return exit code 0 with no output when no real Python is installed.) If an executable path is found, runs `subprocess.run([path, '--version'], capture_output=True, text=True)` to confirm and extract the version string from `result.stdout` (or `result.stderr` if stdout is empty, as some Python builds write version to stderr). Reports "OK: Python X.Y.Z" using the first found. Reports "ERROR: python not found in PATH" only if both `shutil.which` calls return `None` — does not attempt to auto-fix.
- [ ] AC5: `gaai_deliver.py --doctor` checks for `status: in_progress` backlog entries whose `started_at` is older than `GAAI_STALENESS_THRESHOLD` seconds. `GAAI_STALENESS_THRESHOLD` is a module-level constant in `gaai_deliver.py` with default value `15000` (seconds; derived as 14400s nominal delivery timeout + 600s buffer). `started_at` is read from the `started_at:` field in `active.backlog.yaml` as an ISO 8601 UTC datetime string (e.g., `2026-05-12T03:00:00Z`); staleness is computed as `datetime.now(timezone.utc) - datetime.fromisoformat(started_at.replace('Z', '+00:00')) > timedelta(seconds=GAAI_STALENESS_THRESHOLD)`. If `active.backlog.yaml` cannot be read or parsed, the check reports "WARN: could not read active.backlog.yaml for staleness check: {reason}" and skips without affecting exit code. If a `started_at` field is absent (value is `None` in the parsed YAML) or if parsing raises any exception — `AttributeError` when calling `.replace()` on `None`, `ValueError` for malformed ISO 8601 strings, or any other exception from `datetime.fromisoformat()` — that entry is skipped with "WARN: could not parse started_at for {story_id}, skipping staleness check". The exception guard wraps the full expression `datetime.fromisoformat(started_at.replace('Z', '+00:00'))` including the `.replace()` call, to catch both `None` values and malformed strings in a single handler. For each confirmed stale entry: resets to `status: refined` via `_set_backlog_status(story_id, 'refined')` (authorized by DEC-2026-05-12-02) and reports "Fixed: reset {story_id} from in_progress to refined". If none: reports "OK: no stale in_progress stories".
- [ ] AC6: `gaai_deliver.py --doctor` runs two separate headless mode readiness checks under a "headless readiness" grouping: (a) reads the `CLAUDE_SUBPROCESS_ARGS` constant from the running module and verifies `'--dangerously-skip-permissions'` is present — reports "OK: --dangerously-skip-permissions present" or "ERROR: --dangerously-skip-permissions missing from CLAUDE_SUBPROCESS_ARGS — edit CLAUDE_SUBPROCESS_ARGS in gaai_deliver.py and add '--dangerously-skip-permissions'" (ERROR, not WARN, because missing this flag will cause every headless invocation to hang; operator guidance is required because this is a source-code constant that cannot be auto-fixed); (b) reuses the result of AC1's `skipDangerousModePermissionPrompt` check (no second file read is performed) and emits a distinct labeled line reflecting the post-fix state: if AC1 auto-fixed the key or found it present, AC6(b) reports "OK: skipDangerousModePermissionPrompt set (headless readiness)"; if AC1 reported ERROR (key absent and could not be added), AC6(b) reports "ERROR: skipDangerousModePermissionPrompt not set (headless readiness)". This is a separate output line from AC1's output, labeled to show it is being checked in the context of headless readiness, and counted as a separate check in the `doctor: N checks ...` summary. The total summary check count therefore includes both the AC1 settings check and this AC6(b) re-report as two separate items.
- [ ] AC7 (error-handling): `gaai_deliver.py --doctor` exits 0 if all checks pass or all failures were auto-fixed. Exits 1 if any check reports ERROR. WARN items do not affect exit code.
- [ ] AC8: All `--doctor` output is ASCII-only — no Unicode, no emoji. Each check outputs one line: `[OK|Fixed|WARN|ERROR]: <description>`. Final summary line: `doctor: N checks, N fixed, N warnings, N errors`.

## Notes

- `--doctor` is always safe to run repeatedly — all auto-fixes are idempotent
- `--doctor` is invoked automatically by `--unattended` before delivery starts (E07S04 wires this in)
- The `in_progress → refined` reset in AC5 is authorized by DEC-2026-05-12-02 as a governed recovery operation — it does not amend the GAAI lifecycle state machine. Only `--doctor` (AC5) and the E07S02 startup orphan check (AC6) may perform this reset.
- AC6(a) is elevated to ERROR (not WARN) because `--dangerously-skip-permissions` missing from `CLAUDE_SUBPROCESS_ARGS` is a non-auto-fixable configuration error that will cause every headless invocation to hang on an interactive permissions prompt. Because this cannot be auto-fixed, the ERROR message must include operator guidance: "edit CLAUDE_SUBPROCESS_ARGS in gaai_deliver.py and add '--dangerously-skip-permissions'" — giving the operator a clear resolution path before they retry `--unattended`.
