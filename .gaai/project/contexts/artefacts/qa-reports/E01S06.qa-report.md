## MMTH Risk Assessment

**Status: Known risk, appropriately handled — does NOT block PASS.**

- yfinance `^MMTH` is confirmed unavailable (HTTP 404). The `fetch_mmth_yfinance()` function raises `RuntimeError` immediately when no data is returned.
- The EODData fallback (`fetch_mmth_eoddata()`) raises a clear `RuntimeError` with actionable instructions if `EODDATA_API_KEY` is not set:
  > "MMTH data source unavailable: yfinance does not provide ^MMTH data. EODData fallback requires the environment variable 'EODDATA_API_KEY' to be set. Register at https://eodhistoricaldata.com/ (free tier available), then set EODDATA_API_KEY=<your_key> and retry."
- All tests are fully isolated from live network calls via `_spx_df` and `_mmth_series` injection parameters in `load_historical()`. No test touches the real MMTH fetch path.
- AC1 is satisfiable in production once `EODDATA_API_KEY` is configured. The error surfacing is explicit and actionable — this meets the spirit of AC6 for the MMTH case.
- **Risk logged for handoff:** The EODData free tier must be confirmed to provide 252+ trading days of MMTH history before production deployment.
