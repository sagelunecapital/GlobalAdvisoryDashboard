## AC Evaluation

### AC1 — ≥52 weekly / ≥12 monthly observations fetched and persisted

**PASS.**

- `_series_to_records()` in `macro_historical_load.py` enforces `MIN_DAILY_ROWS=52` and `MIN_MONTHLY_ROWS=12`. If a series falls below the threshold after `dropna()`, it raises `RuntimeError` before the record is persisted, ensuring the minimum is never violated silently.
- `fetch_breakeven()` independently enforces `len(raw) < 52` → RuntimeError.
- `fetch_yoy_series()` and `fetch_mom_series()` enforce `len(yoy/mom) < 12` → RuntimeError.
- `fetch_direct_monthly()` enforces `len(raw) < 12` → RuntimeError.
- `test_minimum_observation_counts` in `test_macro_historical_load.py` verifies ≥52 rows per daily indicator and ≥12 rows per monthly indicator in the database after a full load.
- All 32 indicator IDs are stored: 7 inflation, 7 yields, 7 currencies, 5 futures, 6 growth.
- GDPNow stores a single point (current quarter estimate) — the AC says "given initial setup, the system fetches at least 52 weekly / 12 monthly observations for all 32 indicators." The GDPNow exception for single-point storage is structurally built into the orchestrator (`test_minimum_observation_counts` explicitly checks `count >= 1` for GDPNOW), and this is consistent with AC3 which defines GDPNow as a single current-quarter estimate. No ambiguity: AC1 minimum-observation requirement applies to series-based indicators; GDPNow is a point estimate (AC3 governs).

### AC2 — Monthly indicators store most recently released value (no forecast/interpolation)

**PASS.**

- All 10 monthly indicators use FRED REST API, which exclusively returns officially released historical values. FRED does not serve forecasted or interpolated data.
- YoY computation: `((raw / raw.shift(12)) - 1) * 100` followed by `.dropna()` — dropna removes the first 12 rows where the 12-month lag is unavailable, not the most recent released value. The tail of the series is always the latest release.
- MoM computation: same pattern (`shift(1)`), dropna removes only the first row.
- Direct monthly (ISM Mfg, UNRATE, NFP): passed through without transformation; most recent released value is last element.
- ISM Services: FRED series `NMFCI` with month-end conversion — same pass-through.
- Retail Sales: `RSAFS` (advance retail sales) — official monthly release, no interpolation.
- FRED series IDs: `CPIAUCSL`, `CPILFESL`, `PCEPI`, `PCEPILFE`, `PPIACO`, `NAPM`, `UNRATE`, `PAYEMS`, `RSAFS`, `NMFCI` — all are released official series.

### AC3 — GDPNow fetched from Atlanta Federal Reserve Bank GDPNow page

**PASS.**

- `gdpnow.py` uses endpoint: `https://www.atlantafed.org/-/media/documents/cqer/researchcq/gdpnow/GDPNow-model-output.xlsx`
- This is the Atlanta Fed's Excel file for GDPNow model output — the authoritative source specified in the AC.
- The function parses the Excel file, identifies the GDPNow/Nowcast column, sorts by date, and returns the most recent row as `(date_str, estimate_value)`.
- No FRED is used for GDPNow.
- Tests confirm the Excel parsing path with mocked responses.

### AC5 — Subsequent refresh appends only; no overwrites or deletions

**PASS.**

- `macro_append.py` uses `INSERT OR IGNORE` exclusively. The INSERT_SQL constant is `"INSERT OR IGNORE INTO macro_indicators (indicator_id, date, value, fetch_timestamp) VALUES (?, ?, ?, ?)"`.
- No `UPDATE` or `DELETE` SQL exists anywhere in `src/macro/` (verified by source scan).
- Composite primary key `(indicator_id, date)` on the table enforces uniqueness — a duplicate `(indicator_id, date)` pair is silently ignored, not overwritten.
- `test_idempotency_run_twice_no_extra_rows` verifies row count is identical after a second full load with the same data.
- `test_append_partial_duplicate` verifies original value is preserved when a duplicate (indicator_id, date) is submitted.

### AC6 — Failed indicator surfaces visible error; others continue

**PASS.**

- `_persist_series()` wraps each indicator in `try/except Exception`: on failure, `results[indicator_id] = f"error: {e}"` and `logger.error(...)`.
- If a fetch produces no series (e.g., yfinance ticker missing from batch result), `results[indicator_id] = "error: <id> not available"` is set directly.
- Errors are visible in the `results` dict returned by `load_macro_historical()` — every indicator ID maps to either `"ok"` or `"error: <message>"`.
- `test_one_failure_leaves_31_stored` verifies 31 indicators succeed when one (GDPNow) is made to fail.
- `test_per_indicator_error_isolation_multiple_failures` verifies 27+ indicators succeed when 5 are failed simultaneously.
- The function only raises `RuntimeError` when ALL indicators fail (`ok_count == 0`).

### AC7 — GDPNow failure isolated; does not block other indicators

**PASS.**

- GDPNow is fetched in its own `try/except RuntimeError` block (Step 6 in the orchestrator), completely separate from the per-indicator `_persist_series` loop.
- On failure: `results["GDPNOW"] = f"error: {e}"` and `logger.error(...)`. No re-raise.
- Execution continues; the function returns the full results dict with all other indicators.
- `test_one_failure_leaves_31_stored` uses a NaN GDPNow value to trigger GDPNow failure and confirms 31 others succeed.
- The GDPNow error is surfaced via: (1) `results["GDPNOW"]` string, and (2) `logger.error()` call.

### AC8 — Records include indicator_id, date, value, fetch_timestamp

**PASS.**

- Schema definition (`macro_schema.py`): all four columns are `NOT NULL`. SQLite enforces this at insertion time.
- `fetch_timestamp` is set once at the start of `load_macro_historical()` as `datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ")` and passed to every record.
- `_series_to_records()` constructs tuples as `(indicator_id, date_str, float(val), fetch_timestamp)`.
- `test_fetch_timestamp_non_null_on_all_rows` queries `WHERE fetch_timestamp IS NULL` and asserts 0 rows.
- `test_macro_schema_columns` verifies exactly 4 columns: `indicator_id, date, value, fetch_timestamp`.

### AC9 — Consistent named indicator IDs; schema directly queryable for E03S02

**PASS.**

- All 32 indicator IDs are defined in `ALL_INDICATOR_IDS` in `macro_historical_load.py`. The list is the authoritative canonical set.
- Schema: `macro_indicators(indicator_id TEXT, date TEXT, value REAL, fetch_timestamp TEXT)` with composite PK `(indicator_id, date)`.
- E03S02 can compute rolling mean and std dev with: `SELECT date, value FROM macro_indicators WHERE indicator_id = ? ORDER BY date` — no joins, no format transformation.
