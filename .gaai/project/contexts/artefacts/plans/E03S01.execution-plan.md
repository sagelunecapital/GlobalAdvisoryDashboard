## Approach Rationale

**FRED access (DP1):** Use `requests` direct REST calls — already a project dependency,
consistent with `mmth.py` EODData pattern, avoids adding `fredapi` or `pandas_datareader` for
only 2–8 series.

DXY (DX-Y.NYB), and futures (GC=F, CL=F etc.); FRED for US 2Y yield (DGS2), EFFR, all
inflation series, and monthly growth indicators; Atlanta Fed for GDPNow. See evaluation for
full source map.

**GDPNow (DP3):** Atlanta Fed Excel endpoint via `requests` + `pandas.read_excel()`;
`openpyxl` added to requirements.txt.

**Schema (DP4):** Normalized `macro_indicators` table with composite PK `(indicator_id, date)`,
supporting INSERT OR IGNORE idempotency (AC5) and direct z-score queries in E03S02 (AC9).

### Step 5 — fred.py (Daily FRED Series)

Fetches: T5YIE, T2YIE, DGS2, EFFR, DGS3MO (and any other daily FRED series).

Function signature:
```python
def fetch_fred_series(
    series_id: str,
    observation_start: str,  # YYYY-MM-DD
    observation_end: str,    # YYYY-MM-DD
    api_key: str,
) -> pd.Series:
    """
    Fetch a FRED time series by series_id.
    Returns pd.Series with DatetimeIndex and float values.
    FRED returns '.' for missing observations — these are dropped.
    Raises requests.HTTPError on API failure.
    Raises RuntimeError if FRED_API_KEY is not set or series returns no data.
    """
```

FRED API endpoint:
```
GET https://api.stlouisfed.org/fred/series/observations
  ?series_id={series_id}
  &api_key={api_key}
  &file_type=json
  &observation_start={observation_start}
  &observation_end={observation_end}
  &sort_order=asc
```

Error handling:
- Missing `FRED_API_KEY` env var → `RuntimeError` with setup instructions
- HTTP non-2xx → `requests.HTTPError` propagated to caller
- FRED `value == "."` entries → filtered out (missing data)
- Empty result after filtering → `RuntimeError` with series ID in message

### Step 6 — monthly_indicators.py

Fetches all monthly FRED series: CPIAUCSL, CPILFESL, PCEPI, PCEPILFE, PPIACO, PAYEMS,
NAPM, NMFCI, UNRATE, RSAFS.

**YoY computation (CPI, Core CPI, PCE, Core PCE, PPI):**
```python
def compute_yoy_pct(series: pd.Series) -> pd.Series:
    """
    Compute year-over-year % change from a monthly level series.
    ((current_month / same_month_12_periods_ago) - 1) * 100
    Requires at least 13 monthly observations. Returns Series from month 13 onward.
    """
```

Fetch minimum history: 24 months to produce 12 valid YoY observations (AC1 requires ≥12
monthly observations for monthly-released indicators).

**MoM computation (Retail Sales):**
```python
def compute_mom_pct(series: pd.Series) -> pd.Series:
    """
    Compute month-over-month % change.
    ((current / prior) - 1) * 100
    """
```

**ISM series ID verification guard:**
The module must include a runtime check: if fetching NMFCI returns no data, raise a
`RuntimeError` with a message directing the delivery agent to check the FRED series ID for
ISM Non-Manufacturing. This guard must fire before any data is written.

**Monthly date convention:** FRED returns monthly observations with dates on the first of the
month (e.g., `2024-03-01` for March 2024). The fetch module converts these to the last day of
the reference month using `pd.tseries.offsets.MonthEnd(0)`. This is the `date` stored in
`macro_indicators`. Rationale: the last day of the month is the unambiguous reference period
end date and avoids confusion with "as of first of month" timestamps.

### Step 7 — gdpnow.py

Atlanta Fed GDPNow Excel endpoint:
```
https://www.atlantafed.org/-/media/documents/cqer/researchcq/gdpnow/GDPNow-model-output.xlsx
```

Function signature:
```python
def fetch_gdpnow() -> tuple[str, float]:
    """
    Fetch the most recent GDPNow estimate from the Atlanta Fed.
    Returns (date_str, estimate_value) for the most recent available estimate.
    date_str is ISO-8601 (YYYY-MM-DD) of the estimate release date.
    Raises RuntimeError if the endpoint is unavailable or parsing fails.
    This error is ISOLATED — caller must not let it block other indicators.
    """
```

Implementation notes:
- Use `requests.get(url, timeout=30, headers={"User-Agent": "Mozilla/5.0 ..."})` — Atlanta Fed
  may reject requests without a browser-like User-Agent
- Parse with `pd.read_excel(BytesIO(response.content), sheet_name=0)`
- The relevant columns are typically: a date column and a "GDPNow" or similar value column
- Extract the last non-null row as the current estimate
- If the Atlanta Fed changes the column layout, the parse step must fail loudly with the
  actual column names in the error message — no silent wrong-column extraction

AC7 compliance: `fetch_gdpnow()` raises `RuntimeError` on any failure. The caller in
`macro_historical_load.py` catches this exception, logs it as a visible error, and continues
with all other indicators.

**Spread computation (within this module):**

### Step 9 — macro_historical_load.py

This is the orchestrator. It coordinates all fetch modules with per-indicator error isolation
and calls `macro_append.py` to persist results.

```python
def load_macro_historical(
    db_path: str,
    fred_api_key: str | None = None,
    _overrides: dict[str, pd.Series] | None = None,  # for testing only
) -> dict[str, str]:
    """
    Fetch and persist all 32 macro indicators.

    Returns a summary dict: {indicator_id: 'ok' | 'error: <message>'}.
    Successful indicators are persisted even if some fail.
    Errors are collected and logged — not raised until the end (summary mode).

    Args:
        db_path: SQLite database path.
        fred_api_key: FRED API key. If None, reads FRED_API_KEY env var.
        _overrides: (testing only) pre-built Series dict bypassing live fetch.

    Raises:
        RuntimeError: only if zero indicators were successfully fetched and stored.
                      Individual indicator failures do not raise — they appear in the
                      returned summary dict.
    """
```

**Per-indicator isolation pattern (AC6, AC7):**
```python
errors = {}
results = {}
for indicator_id, fetch_fn in fetch_registry.items():
    try:
        series = fetch_fn()
        # persist via macro_append
        results[indicator_id] = "ok"
    except Exception as e:
        errors[indicator_id] = str(e)
        print(f"[ERROR] {indicator_id}: {e}", file=sys.stderr)  # visible error (AC6)
```

After all fetches: if `errors` is non-empty, print a summary of failed indicators to stderr.
If all indicators failed: raise `RuntimeError`.

**Schema initialization:** `macro_schema.create_macro_schema(db_path)` is called once before
the fetch loop.

**fetch_timestamp:** Generated once per `load_macro_historical()` call as
`datetime.utcnow().strftime("%Y-%m-%dT%H:%M:%SZ")`. All records within one load run share
the same timestamp.

**Minimum observation validation:**
After persisting, validate that each indicator_id has the minimum required observations:
- Daily indicators: ≥52 records (52 weeks of daily closing values)
- Monthly indicators: ≥12 records (12 monthly observations)

If any indicator has fewer than the minimum, log a warning (not an error) with the actual count.

## Edge Cases

### AC1 — Minimum Observation Depth

- **Happy path:** All daily indicators return 52+ weekly data points (1+ year of daily data);
  all monthly indicators return 12+ monthly observations.
- **Boundary (daily):** yfinance returns data for trading days only; 52 weeks ≈ 260 trading
  days for daily series. Assertion: `len(series) >= 52` not `>= 260` — the AC requires 52
  *weekly observations*, not 260 daily rows. Clarification: store daily rows, verify
  `count(indicator_id) >= 52` (at least 52 distinct date rows, which represents weekly
  coverage). For daily series with 260 rows, this is easily satisfied.
- **Boundary (monthly):** Monthly series must have ≥12 distinct monthly dates stored.
- **Error path:** yfinance returns empty DataFrame for a ticker → per-indicator error logged,
  remaining indicators continue.

### AC2 — Monthly Indicators Use Most Recently Released Value

- **Happy path:** FRED returns the latest published monthly observation for CPIAUCSL etc.
  FRED does not return future/forecast values — the last observation is always the most
  recently released.
- **Boundary:** FRED may return a preliminary value for the most recent month that is later
  revised. E03S01 stores what FRED provides at fetch time — revisions are not backfilled
  (this is a data freshness characteristic, not a bug).
- **Error path:** If the most recent FRED observation has `value == "."` (not yet released),
  the previous month's value is used as the most recent released value. This is handled by
  the `"."` filter in `fred.py` — only non-missing values are returned.

### AC3 — GDPNow from Atlanta Fed

- **Happy path:** Excel downloaded, parsed, most recent row extracted, stored as
  `(GDPNOW, <date>, <value>, <timestamp>)`.
- **Boundary:** GDPNow is updated multiple times per week. On re-fetch, the new estimate
  has a new date — INSERT OR IGNORE will add the new row without modifying the prior one.
  If the date has not changed (same release day re-fetched), INSERT OR IGNORE skips it.
- **Error path (AC7):** Atlanta Fed endpoint unavailable (HTTP error, timeout, parse failure)
  → `RuntimeError` raised by `fetch_gdpnow()`, caught by orchestrator, logged as visible error,
  other 31 indicators continue unaffected.
- **Error path:** Excel column layout changed → parse step fails with column names in error
  message; error isolated to GDPNOW.

### AC4 — Continuous Front-Month Futures

- **Happy path:** yfinance `GC=F`, `CL=F`, `HG=F`, `NG=F`, `ZC=F` return continuous price
  series; stored at the Close price per day.
- **Boundary:** Roll gap dates — on contract roll dates, the series may show a price
  discontinuity. These are stored as-is. No adjustment applied (out of scope).
- **Error path:** A futures ticker returns no data (e.g., `NG=F` experiencing data issues)
  → per-indicator error, remaining futures continue.

### AC5 — Append-Only, No Overwrites

- **Happy path:** Re-running `load_macro_historical()` after initial load → all existing
  `(indicator_id, date)` pairs are skipped by INSERT OR IGNORE; only genuinely new dates are
  appended.
- **Boundary:** Re-fetch on the same day with the same data → row count unchanged, no error.
- **Boundary:** Re-fetch after a new FRED release → one new monthly row per indicator
  appears; existing rows unchanged.
- **Error path:** No error path — INSERT OR IGNORE is structurally safe; database write
  failures (disk full, locked) raise `sqlite3.OperationalError` which propagates as a
  per-indicator error.

### AC6 — Per-Indicator Error Isolation

- **Happy path:** All 32 indicators fetched successfully; summary dict shows all "ok".
- **Boundary:** 1 indicator fails → 31 stored, 1 error logged to stderr, summary shows
  `{failed_id: "error: ..."}`.
- **Error path:** FRED API key missing → all FRED-sourced indicators fail with the same
  `FRED_API_KEY not set` error; yfinance indicators still proceed and are stored.
- **Worst case:** All indicators fail → orchestrator raises `RuntimeError` after the loop.
  DB may be empty (if first run) or contain prior data (if re-run). No partial corruption.

### AC7 — GDPNow Source Unavailable

- **Happy path:** See AC3 happy path.
- **Error path:** Any exception from `fetch_gdpnow()` is caught at the orchestrator level:
  ```
  [ERROR] GDPNOW: Atlanta Fed endpoint unavailable (HTTP 503). Other indicators unaffected.
  ```
  The error must be visible (stderr or print). GDPNow's failure does NOT prevent the other
  31 indicators from being stored.

### AC8 — fetch_timestamp in Every Record

- **Happy path:** Every inserted row has a non-null `fetch_timestamp` in ISO-8601 UTC format.
- **Boundary:** Two runs on the same day → both rows (if dates differ) have distinct
  fetch_timestamps. If same `(indicator_id, date)` — INSERT OR IGNORE skips, original
  fetch_timestamp preserved.
- **Error path:** `fetch_timestamp` is set in the orchestrator before calling append — it
  cannot be null unless the orchestrator itself has a bug. Test verifies non-null.

### AC9 — Consistent Indicator IDs for Downstream z-Score

- **Happy path:** All 32 indicators stored with the canonical IDs from the source map. E03S02
  queries `SELECT date, value FROM macro_indicators WHERE indicator_id = 'US_10Y_YIELD'`
  without any format transformation.
- **Boundary:** indicator_id values are validated against the canonical list before insert —
  no unexpected IDs are persisted.
- **Error path:** A fetch module returns data under a wrong ID → unit tests for that module
  assert the correct indicator_id is returned.

## Risk Register

| Risk | Severity | Mitigation |
|------|----------|------------|
| ISM Services PMI series ID (NMFCI) not returning current data on FRED | High | Runtime check guard in `monthly_indicators.py` raises explicit error with FRED search instructions. Delivery agent verifies NMFCI before writing persist code. |
| Atlanta Fed GDPNow Excel URL changed or moved | High | Isolated to GDPNOW fetch (AC7). Error message includes full URL. Delivery agent checks the Atlanta Fed GDPNow page if fetch fails. |
| ^IRX not available as 2Y Treasury yield proxy (it is 13W, not 2Y) | High | Already resolved in DP2: 2Y yield sourced from FRED DGS2, not yfinance. ^IRX used only for 3M spread input. |
| FRED API key not set in CI environment | High | CI must have `FRED_API_KEY` set as a secret. If not set, all FRED-sourced indicators fail. Unit tests must mock FRED responses to avoid needing real API key in CI. |
| yfinance rate limiting on batch download of 12+ tickers | Medium | Use `yfinance.download()` with a list of tickers in one call rather than individual downloads — yfinance batches internally. Add 1-second sleep between fetch groups if rate limiting occurs. |
| Atlanta Fed User-Agent blocking | Medium | Use browser-like User-Agent header in `requests.get()`. Document in code comment. If blocked, surface error — do not silently fail. |
| ^CNH=X insufficient history | Low | Fallback to CNY=X with warning logged. Indicator ID remains USD_CNH. |
| Roll gaps in futures data causing downstream anomalies | Low | Out of scope for E03S01. Store as-is and document in code comment. E03S02 will handle if needed. |
| Monthly date normalization (first vs. last day of month) inconsistency | Medium | Enforce month-end date conversion in `monthly_indicators.py`; test verifies all monthly dates are month-end. |
| Turn budget overrun | Medium | Module count is high (7 files + 6 test files). Target: Steps 1–9 in ~70 turns, test writing ~30 turns, CI remediation budget ~40 turns. Total ~140 turns, within 150-turn target. |
| `openpyxl` version conflict with existing pandas | Low | `openpyxl>=3.1.0` is broadly compatible with pandas >=2.0. CI will surface any conflict immediately. |
